1. O que é CI/CD e por que é importante?
- CI/CD é um processo que integra e testa o código automaticamente a cada mudança, reduzindo erros e garantindo maior qualidade no software.

2. Em qual pasta os workflows do GitHub ficam armazenados?
- Na pasta .github/workflows.

3. O que aparece no log do GitHub Actions após a execução?
- O resultado da execução do script main.py.

4. O que acontece se alterar o código e fizer novo push?
- O workflow roda novamente automaticamente.

5. O que acontece se um teste falhar?
- O GitHub Actions marca o workflow como falhou e interrompe o pipeline.
